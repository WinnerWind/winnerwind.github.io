Learn to Linux - 1

Introduction

#Blogs/Tutorials/Linux
Linux may seem really daunting at first, and this guide aims to make Linux easily digestible. If you have never used Linux before, or have never interacted with a computer as we will in this guide, you may want to start here. It is assumed that you have a Linux distribution (distro) running on your system, or on a virtual machine, preferably with a terminal.
I will not be getting into advanced system administration in this guide; that's beyond what I wish to teach you. I want to teach you how Linux can be used on your own personal computer.

Terminology

I don't expect you to mug all of this up right now. Please don't attempt that, because it might hinder your progress, and in my opinion, by the time you are done with this guide, you will have organically learnt most of these terms

Distro

A Distro is the short form for "Distribution". A Linux distribution is just a derivative of Linux.

Kernel

This is what Linux really is. It is the core of the operating system, and every Distro just builds upon the Kernel.

Root / Root User

The Root user is the most powerful user in Linux. All commands that modify the system must come from the Root User.

Users

A user is simply someone who uses a computer. In the context of Linux, it means an account.

The Basics

Linux; It's everything Windows does but better and open source. It may seem a bit daunting at first, but once you get the hang of it, it really is a breeze. We'll only be talking about commands in this guide, as that is the part most users get stuck at.

Formats

All commands follow the following formats

  • command --argument
  • command -a
  • command --linked-argument linked stuff
  • command argument1
    Do note that some programs tend to use different formats. A "linked argument" is an argument that also requires an additional setting, like for example, in Arch Linux, to install, say, nano, you run :
pacman -S nano

Note that running pacman -S will result in an error.

Arguments

Analogy

Lets say that you and your friend need to move some boxes. You have a clipboard having the details of all the boxes and also where exactly to put them. Deciding to split up the effort, you instruct your friend, using the clipboard on where exactly to put them. You say : "Move the red box to the green square"

Similarly, assume we have a command that moves boxes. Lets say that this command is called movebox. How do we tell movebox to move the red box to the green square? For this hypothetical command, you might write something like this.

movebox --color red --to green-square

Note how the command makes no sense without the addition of red after --color and green-square next to --to. So, we can say that --color red --to green-square specifies movebox on what to do. These specifications are called arguments.

Hyphened, single lettered, and word arguments

For most commands, they start with a hyphen (-), two for whole words (--). Not every argument is a linked one, and some can work without further specification, like this one below

firefox --devtools

Also, not every command uses hyphens for arguments, for example

spicetify backup update

or

mount /dev/sda1 /mnt/usb

Single lettered arguments are quite useful to remember and are often short forms of the full word arguments. These will help you save precious time typing. For example, to run a full system update on Arch Linux

pacman -S -y -u

That's quite tedious to write is it not? Sometimes, you can merge these into one hyphen, making it easy to type.

pacman -Syu

Also, sometimes, they're a mix of all or some of the above types!

mount /dev/sda1 /mnt/windows --mkdir

Don't worry if this is a lot of information to digest at once. Don't bother memorizing this part for now. You'll eventually understand the anatomy of a command with time.

Manuals

Most Linux distributions come with a command called man. See a command you don't understand? man comes in clutch! Simply run man <command> and it'll try to pull out as much information as it can on the command. For example, if you want to learn what mount does, you can run

man mount

and learn more about it. Learn more about man by running man man

Directories and files

Directories

List content

Unlike Windows, Linux uses forward slashes (/) to navigate directories. You can see where your terminal is running from by typing pwd which stands for Print Working Directory. Note how it uses forward slashes instead of back slashes.

Your system always gives a short form for where you are at the start of the typing caret. Note that tilda (~) is the short form for /home/<username>

You can see the current directory's contents by running ls, which stands for "List". You can view the current directory in a tabular format by using ls -l. Try using man to figure out what the -l argument does. By default, the output is coloured to show what type it is.

You can also specify the directory you wish to inspect by passing it as an argument to ls. 

ls -l /mnt

Navigation

You can navigate directories by using the cd command, which stands for "Change Directory". If you are currently at ~ (/home/<username>/) and want to navigate to Downloads, you run 

cd Downloads

You can specify a directory with reference to the Root folder (more on that later) by starting where you want to go with a /. So,

cd Downloads

takes you to /home/<username>/Downloads/, but

cd /Downloads

takes you to /Downloads/. More on this later.

You can navigate backwards by running 

cd ..

which takes you back by one directory. For example, if you are in /home/<username>/Downloads/, running cd .. will take you to /home/<username>/

Making Directories

Sometimes, it is vital to make a directory. You can do so with the mkdir command. Simply run mkdir followed by the name of the directory you wish to create. For example, if you want to make a directory named "foobar", you run

mkdir foobar

In case you wish to specify where it is to be made, you can specify that too, by pointing it to a directory instead. If you want to make a directory named "foobar" in /home/<username>/Downloads, you can do so by

mkdir ~/Downloads/foobar

or

mkdir /home/<username>/Downloads/foobar


Note that if your mkdir command is made in a directory whose parent directories don't exist, they will NOT be created.

Note that the directory testdir does not exist in the home directory

Note on Absolute references and Relative references.

An absolute reference is a reference coming from the Root directory. It tells exactly where the file or directory is. Eg. /home/<username>/Downloads
A relative reference is something that changes from place to place. It may use short forms to choose to not specify the root directory at all. Eg. ~/Downloads/ or Downloads/. Note how in the second example, the computer is never told explicitly where exactly Downloads/ is located. However, /Downloads/ tells the computer that it originates from the Root directory.
We will discuss the root directory another time.

Removing Directories

Removing anything can be done with the all powerful rm command, which permanently deletes the file. Simply run rm with the argument r (Recursive) to delete a directory that is not empty. Be careful, as running this on the wrong directory might brick your whole system

rm -r foobar/

Alternately you can use rmdir to remove a directory. This command runs similar to rm and is made specifically for removing directories. It does not need the r option to delete a directory that is empty.

rmdir foobar/

for an empty directory, and

rmdir -r foobar/

for a non-empty directory

Files

All files have a file format, which is defined by a period (.). Windows too uses the same system. Note that changing the file format manually may cause the file to be unreadable, but it can be fixed by reverting it's file format back; the contents of the file are not changed when its format is changed.

Making files

Usually files are made using the touch command. This is a hack, because touch is used to modify file timestamps, but a byproduct of that is that it can be used to make files.

Simply run touch followed by the name of the file you wish.

Note that touch usually cannot differentiate between a file and a directory and may end up modifying the file time stamp of a directory or file.

Removing files

You can remove a file using rm. No need for fancy arguments in most cases. For example, if you want to remove the file named foobar, run

rm foobar
Files removed this way CANNOT be recovered

Moving Files

You can move a file with the mv command, which means "Move". Simply supply the file location of the source file and the file location of the destination file. If you want to move the file foo from your home directory (/home/<username> or ~) to your Downloads, you can do it like so.

mv ~/foo ~/Downloads/
You do not need to supply the destination file name, as it is already implied.

You can move multiple files by just adding more files to the arguments like so.

mv file1 file2 file3 filen ~/Downloads/

Renaming files

Using the mv command, you can also rename files. Similar to the previous ones, a directory may be specified if the file is not located in your working directory.
