Setting up an IME

People who wish to use IMEs to type in non-English languages will need to read this guide. For this guide, it assumed that you wish to install a Japanese IME (As it's the most popular type), however other languages are compatible too. Read this guide for other languages.
Even though I say "For XYZ environment, they can be used interchangably. Eg. Using FCITX5 in GNOME, or using IBUS in KDE Plasma, etc.

IBUS for GNOME based Desktop Environments

  1. Run yay -S ibus ibus-mozc
  2. If it asks for whether you'd like Mozc or Mozc UT, select Mozc UT if you want the UT enhanced dictionary and select Mozc if you want the standard Google dictionary.
  3. Give it some time to compile. It takes a long time and a lot of power.
  4. After that's done, run ibus-setup. Go to Input Method then add both your English - English setup and another one (In the Japanese Section) called Mozc：あ If it does not appear, restart your system
  5. Add an autostart entry (either through Budgie Desktop settings, or writing a file using the template below to ~/.config/autostart by using nano) with the command ibus-daemon -rxRd
  6. If you can't create an autostart entry, run nano ~/.config/autostart/ibus_daemon.desktop and copy paste this. (Even the [Desktop Entry] at the start!)
[Desktop Entry]
Type=Application
Name=Ibus Daemon
Description=Starts the Ibus Daemon
Exec=ibus-daemon -rxRd
  1. Modify /etc/environment using nano Use sudo
  2. Copy paste this and put it at the bottom.
GTK_IM_MODULE=ibus
QT_IM_MODULE=ibus
XMODIFIERS=@im=ibus
MOZC_IBUS_CANDIDATE_WINDOW=ibus
  1. Go to your system settings (Gnome settings or Budgie Control Center) then in the keyboard tab, add Mozc:あ here.
  2. Mozc should just work now! Be sure to restart your system to run the autostart entry.
  3. If you wish to modify Mozc settings, a small pop-up should appear in one corner of the screen. Alternately, in ibus-setup, under the Input Method tab, you can click preferences when Mozc is selected to see more settings. You can also theme it in ibus-setup

FCITX5 for QT based Environments (KDE Plasma)

FCITX5 is admittedly easier to install than IBUS, and does not need long compile times as it is mostly pre-compiled. It also runs like a traditional IME, and sits in your system tray, so it does not need you to add a separate, virtual keyboard layout.

  1. Install FCITX5 by installing fcitx5-im. Be sure to get every member in it's group, as it comes in handy.
  2. Install the backend by installing fcitx5-mozc.
  3. Add this to your /etc/environment Use sudo
GTK_IM_MODULE=fcitx
QT_IM_MODULE=fcitx
XMODIFIERS=@im=fcitx
  1. Restart your computer.
  2. If you're in KDE Plasma, an option called "Input Methods" or "FCITX" may appear as a tab within the settings. It has a penguin logo. You can add MOZC there by selecting it from the right pane and adding it into the left pane. (If you don't have KDE Plasma, an app called FCITX 5 Configtool should appear as a standalone app)
  3. Modify keybind settings in the "Global Options" tab.
  4. Modify MOZC settings and theme settings in the "Addons" tab. (Input Method -> Mozc and UI -> Classic User Interface respectively)
  5. FCITX5 should work as intended now.
  6. If you wish to change the Kana settings (Hiragana/Katakana, etc.), right click the tray icon and select your desired mode.

Note.

If you use a DE that runs under Wayland (KDE Plasma for Wayland, Hyprland, etc.) You will get a notification with extra instructions to make it work.
