Learn to Linux - 2

Introduction

Welcome back! In this chapter, we will learn more about the nitty-gritty about Linux and how it works under the hood. Don't worry, we won't do anything too advanced here. In case you haven't, read the previous chapter before jumping into this one.

Terminology

Again, don't memorise all of these, just refer to these if you need to.

Root Folder

A root folder is where your Linux installation originates from. It can be thought of as the "roots" of a tree. It's located in / and every directory is a sub-directory of it.

Groups

Every user belongs to one or more groups, which is also used for organising large clusters of users (Eg. A big organisation might need different groups for separating teams, letting certain users use certain printers and most of all, to limit what commands they can run on their system), or to prevent users from certain groups from accessing certain files and folders (Eg. Team A accessing files of Team B would be problematic, so groups prevent Team A from accessing Team B's files).

Content

Folder Structure

Linux comes with a rather unique structure to directories. You might have noticed that the pwd starts with the /. This is called the "root" folder. Like a tree, this directory holds everything in your system. Running ls / -l to see it's contents, you can see quite a few directories.

Most, if not, all Linux systems come by default with such a file directory structure. It's not important to know what each and every directory does, but for the sake of discussion, let's talk about a few. Note that the below diagram only shows the most important ones.

Home (/home/)

The Home directory, located under /home/ contains all day to day files for all users. Usually a specific folder pertaining to your username will exist which contains all your files. All files here also belong to your user and can be freely modified and deleted. You can check this by running ls -l ~ and comparing it with the result of ls -l /. Note that one tells ls to list in a tabular format your home directory, and one tells it to list in a tabular format the root directory. ~ is a shortcut for telling Bash to look into your own home folder.

Bin (/bin/)

Although the name might make you imagine a trash bin, like Windows' Recycle bin, it actually stands for "Binary". Here the code for all your programs are stored, and even the code for commands exist here. Usually, it's not needed to touch, but is good to know.

Note on Symlinks

You may have noticed that when inspecting the root directory, it actually shows an arrow pointing to /usr/bin/. This is a symlink. It's like a shortcut showing that /bin/ actually points to /usr/bin/. We shall learn about this later.

Mount (/mnt/)

The /mnt/ directory is a special directory that is used for intense system work involving mounting partitions. It is usually touched by the user only. We'll learn about mounting soon.

Boot (/boot/)

This directory contains special files that help Linux boot properly. It's also where your boot-loader is stored. If you use an UEFI system, it's also where your other operating system's boot-loaders stay, thanks to mounting.

Etc (/etc/)

This directory, like the name would imply, contains all other files your system may need, mainly configuration files. It can be thought of as the nerve center of your Linux machine, and contains various vital configuration files required for the system to run.

Users and Groups

As we learnt in the first chapter, whenever you use the system, you're doing so as a user. Imagine if every user was able to access every other user's files and had infinite control over a system. That would be quite problematic, so Linux also says that every user is part of a "group".
It is through the group that you get permission to do certain things, and very rarely are permissions given to single users. In fact, when you list a directory, you can also see what group it belongs to.

Do remember that a user can belong to multiple groups. You can check this out with the id command

id


Alternately, you can also check out just the groups you are in, with 

groups

Adding/removing from/to groups

Any user can arbitrarily be added to and removed from groups.
The specific command may vary on your distribution

List all groups

  • id <user> to list all groups a user is in with their unique identifier. UID is for user ID and GID shows the group ID.
  • groups <user> shows all groups a user is in.

Adding a user to a group

You will require superuser permissions to do these.

  • usermod -aG <group name> <user>

Command Shortcuts

This is a short one, and in a way it's a small break from what we've been doing so far.
There are several key-binds most terminals use for various functions. Two of the most useful ones are CTRL+C and CTRL+Z, along with the usual commands for copying and pasting, which are different due to how the Linux Terminal works (They are CTRL+SHIFT+C and CTRL+SHIFT+V respectively, though this may depend on your terminal emulator too!).

  • CTRL+C is used to kill a command. It doesn't kill it by forcefully stopping it, rather it sends a signal to the command to gracefully stop. This is an important distinction to make, because some commands will continue to execute (usually clean up what they were doing) before stopping.
  • CTRL+Z is used to pause a command while it's executing. In this state, you can run other commands too, and pause them too, making command 'chains'. You can also run bg while a job is paused to make it run in the background, and allow you to close the terminal as it executes. This is quite handy for commands that will take a while to execute. Typing fg instead runs the command as it was doing before, ensuring that the terminal stays open as the command is executed.
    In general, these two shortcuts handle 90% of daily Linux use.
