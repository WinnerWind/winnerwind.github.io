description: Have you ever felt that installing Arch Linux is hard? Well don't fret! Installing it is actually super easy. Read this guide and you'll be up and running in no time!
author: WinnerWind

Introduction

Installing Arch Linux may seem hard, but it's actually super easy. Don't let the pressure of "Arch is the hardest operating system in the world!" get to you. Here's how to install Arch.
You can install arch by following this wiki too, however it is pretty convoluted Arch Installation Guide
You can also install Arch by using archinstall Learn more

Guide

0. Set-up

0.1 Partition your disk

  1. Using Minitool Partition Wizard, Partition your disk into such based on whether you wish to dual boot (Be able to use Windows and Arch Linux in the same system) or replace Windows with Arch Linux. Skip to point number 4 if you're not dualbooting.
  2. Check if your system is EFI or BIOS based.
  3. If you're dualbooting, Partition it into 4 parts by shrinking Windows' partition. Here's how they should be if you're on an EFI system (The order of the partitions is mostly irrelevant)
  • An EFI system partition of 1GB
  • A Linux SWAP partition of atleast 4GB (More is better, however don't exceed the amount of your RAM. Check the amount of your RAM in the Task Manager)
  • An EXT4 partition of atleast 32GB. Here is where all your files go
  • Your Windows Partition
  • (All other partitions)
  1. If you're on an BIOS Partition, make the exact same partitions except the EFI partition.
  2. Don't forget to format them as such, but it doesn't really matter in this stage. Be careful not to accidentally format your Windows' partiton! If you have installed other Linux based operating systems, you would know that usually we leave the partition unformatted, but Arch's installer cannot read unformatted/unallocated partitions. As such, it is needed to format it. The exact formatting used does not matter, however they must be distinct so that they don't merge into one partition
  3. If you're not going to dualboot, don't make the EXT4 partition. Your Windows' partition will be overwritten in later steps. This will irrevocably delete all data on your Windows' partition and replace it with Linux.

0.2. Create a installer USB.

Things you'll need : A USB with at least 8GB storage. All files on this USB will be irrevocably deleted in this section.

  1. Regardless of operating system, get the app Balena Etcher. It's a GUI based tool that lets us create an installer USB.
  2. Visit this page and under the section "BitTorrent Download", download the Torrent. You will need a torrent client like uTorrent.
  3. Open Balena Etcher and plug in your USB. Select the .iso file that you downloaded using torrent and flash your USB. Be careful in selecting the correct USB, as doing so may delete all the files on a used USB or reformat your main drive
  4. After the flash is done, you have an installer USB. Congratulations!

0.3. Boot into your installer USB

This part gets a bit tricky because it depends from system to system, however the general instructions stay the same.

  1. Disable Secure Boot
  2. Reboot your system.
  3. When the manufacturer logo is shown during boot up, be sure to look for text that reads on the lines of "Press (button) to enter BIOS Menu". Some systems require a full shut down to show this text
  4. If the text is not shown, keep pressing buttons until a beep sound is heard and/or the BIOS menu opens.
  5. Select "Choose another boot medium" This step varies from system to system heavily. Please refer to your motherboard's manual
  6. Select Arch Linux Install Medium
  7. Press Enter for Boot into Arch Linux Arch updates all the time, so it may be called something else. Just go with the first option. The second option usually includes a screen-reader if you're visually impaired.

1. Install

Now that we have booted into Arch, the install begins. After the image is copied to RAM, you should be greeted with a message and a terminal. Do not remove the USB during the install at any point, as it may have unintended side effects and consequences (Mostly not permanent)

  1. If the font is too small (as may be the case for 4K displays), type in setfont ter-132b verbatim to switch to a bigger font. This font is huge, so try to stick to the default font as much as possible

1.1. Connecting to the Internet

  1. If you wish to use Ethernet, plug it in, and skip to point number 10 or else continue. Type in rfkill and make sure that the output looks like this
ID TYPE      DEVICE      SOFT      HARD
 0 bluetooth hci0   unblocked unblocked
 1 wlan      phy0   unblocked unblocked
  1. If you see that wlan has the word blocked under SOFT then administer the command rfkill unblock wlan. If it has the word blocked under HARD then make sure that there is no physical switch that is turning off the WiFi card.
  2. Type in iwctl. This should open an interactive prompt starting with the words [iwd]#. This shows that you're using iwctl.
  3. Type device list.
  4. If the device is turned off or it's adapter is turned off, turn it on using device [name] set-property Powered on and adapter [name] set-property Powered on replacing [name] with the name of the device.
  5. Initiate a scan using station [name] scan
  6. List all networks using station [name] get-networks
  7. Then connect using station [name] connect [name of the network you wish to connect to]
  8. Enter password if any.
  9. Once this is done, press the key combination CTRL+C. This kills the current command. Alternately you can exit by administering exit.
  10. Type in ping google.com. If you get an error like ping: google.com: Temporary failure in name resolution then you may have messed up in the earlier steps. IWCTL is a bit hard to work with, so don't worry about trying again. If it starts pinging, then you'll get periodic messages on screen. This shows that your internet is working. Kill the command using CTRL+C when done. We have officially connected to the internet. This is a very important step.

1.2. Format the partitioned disk.

Now we copy Linux's image into the newly partitioned disk. Be careful during these steps, as missing one can set you back by several more steps or need you to restart from the start.

  1. Type in fdisk -l. You will get an output similar to this.
Sector size (logical/physical): 512 bytes / 512 bytes
I/O size (minimum/optimal): 512 bytes / 512 bytes
Disklabel type: gpt
Disk identifier: 0C8B0FFB-9128-40E5-8869-2A102B375F98

Device             Start       End   Sectors   Size Type
/dev/nvme0n1p1      2048    206847    204800   100M EFI System
/dev/nvme0n1p2    206848    239615     32768    16M Microsoft reserved
/dev/nvme0n1p3    239616 797470719 797231104 380.1G Microsoft basic data
/dev/nvme0n1p4 797470720 965242879 167772160    80G Microsoft basic data
/dev/nvme0n1p5 965242880 973631487   8388608     4G Microsoft basic data
/dev/nvme0n1p6 973631488 975730687   2099200     1G Microsoft basic data
/dev/nvme0n1p7 975730688 976771071   1040384   508M Windows recovery environment
  1. Don't forget which partition is which! The codes in the Device section are important. Be sure to match that with the Type and Size when proceeding.
  2. Use mkfs.ext4 /dev/[code] to format the root partition, where all your files go. replace [code] with the appropriate item from Device. Do NOT mess up the code. Doing so may overwrite the wrong partition, leading to loss of data, especially if you are dualbooting
  3. As an example, in the list I showed earlier, I want to dualboot, so I made the 80GB partition for my files. So, I'll use the command mkfs.ext4 /dev/nvme0n1p4
  4. Set the SWAP partition using mkswap /dev/[code] replacing [code] with the correct Device of the SWAP partition
  5. If you made an EFI system, use mkfs.fat -F 32 /dev/[code] replacing [code] with the correct Device of the EFI system. If not, skip this section.

1.3. Installing essential packages.

Before we can install the packages we need to run Linux, we need to mount our system. Mounting is like opening a door. No files are actually transferred, just a portal is made.

  1. mount /dev/[partition where files are stored] /mnt to mount the partition where the files are stored (Called the Root Partition because that's where the 'root' of the operating system is) to the folder /mnt
  2. You might get the error that /mnt does not exist. Use mkdir /mnt to make the required directory
  3. If you made an EFI partition, run mount --mkdir /dev/[efi partition] /mnt/boot to mount it to /mnt/boot
  4. To use the swap, use the command swapon /dev/[swap partition]. Swap is like a bucket for your RAM. If the RAM gets full, it is sent to the SWAP to make sure that all your programs run smoothly.
  5. Use this command install some packages which will be needed to make the most out of your system. pacstrap -K /mnt base linux linux-firmware nano sudo man grub efibootmgr networkmanager. It installs nano, sudo, man and other needed packages along with the firmware and kernel. More on them later.
  6. Install intel-ucode or amd-ucode based on your CPU manufacturer using pacstrap -K /mnt intel-ucode or pacstrap -K /mnt amd-ucode

1.4. Configuration

Congratulations! You have installed Arch Linux, but it's not quite ready to use. Lets configure it and make it ready to use.

1.4.1 FStab

  1. Generate an FStab file. An FStab file is used to automatically mount certain partitions of the disc on startup. Without it, it would be impossible to boot into Linux. Simply run genfstab -U /mnt >> /mnt/etc/fstab. Edit it using nano /mnt/etc/fstab and check it in case of duplicate entries. BIOS systems should have two entries and EFI systems should have 3. It should look something like this
# Static information about the filesystems.
# See fstab(5) for details.

# <file system> <dir> <type> <options> <dump> <pass>
# /dev/nvme0n1p4
UUID=b541dd69-73ce-4989-94f9-4d210941b047       /               ext4            rw,relatime     0 1

# /dev/nvme0n1p6
UUID=6806-2009          /boot           vfat            rw,relatime,fmask=0022,dmask=0022,codepage=437,iocharset=ascii,shortname=mixed,utf8,errors=remount-ro   0 2

# /dev/nvme0n1p5
UUID=0e5ee460-ec66-4d16-a25e-22d1dd370663       none            swap            defaults        0 0

Fstab can be used to mount Windows on startup if you are dualbooting

1.4.2. Setting the time

  1. Change root into the new system using arch-chroot /mnt. We are now taking control of the newly formed system.
  2. Time is important, especially for a basic system like this! Make sure you set the correct timezone using ln -sf /usr/share/zoneinfo/Region/City /etc/localtime. Replace /Region/City with your Continent and City. Even though I live in Bangalore, I set mine to /Asia/Calcutta as that is where the timezone is located. Also run hwclock --systohc to set the hardware clock. People dualbooting with Windows should check out this guide after finishing the install. Fix dualboot issues regarding time

1.4.3. Setting locales.

  1. Edit /etc/locale.gen using nano. Uncomment en_US.UTF-8 UTF-8 and other locales you might need. A comment starts with a # symbol, so simply remove the # and keep the text untouched. Use the key combination CTRL+W to quickly search the whole file
  2. Your file should now look similar to this.
...
#en_SG.UTF-8 UTF-8
#en_SG ISO-8859-1
en_US.UTF-8 UTF-8
#en_US ISO-8859-1
#en_ZA.UTF-8 UTF-8
...
  1. Run locale-gen.

1.4.4. Final Touches

  1. Set the root password using passwd. You can change this at any time by using the same command when logged in.
  2. Run nano /etc/hostname.
  3. Put in any name you wish here, preferably something with no special characters, and no capital letters. This will be the name devices on your network will see. Your file should look something like this.
    /etc/hostname
yournamehere
1.4.4.1. Setting up GRUB.

GRUB is the boot loader we will be using for our Linux install. It even supports booting into other operating systems. We have already installed GRUB in section 1.3.

  1. Mount the EFI system partition by looking at fdisk -l and mounting it using mount /dev/[efi partition] /mnt You may not be able to mount while using CHRoot. If this is the case, reboot your system and enter into your newly installed Arch system]
  2. grub-install --target=x86_64-efi --efi-directory=/mnt --bootloader-id=GRUB
  3. Run grub-mkconfig -o /boot/grub/grub.cfg to make the config file. If you are not dual booting, then you are done with installing Grub.
  4. Mount the EFI partition belonging to Windows (Often simply labelled EFI System, System Reserved or just SYSTEM) to /mnt/windows
  5. Edit /etc/default/grub using nano. Un-comment the line GRUB_DISABLE_OS_PROBER=false, by removing the # at the start. Use the key combination]``[!!|keyboard|x
  6. If that line does not exist, add it.
  7. Run os-prober
  8. For good measure, run grub-mkconfig -o /boot/grub/grub.cfg again.

1.5. Rebooting into the system

  1. If you are in CHroot, type exit
  2. Type in reboot
  3. Boot into your new system.

2. Post Install.

Congratulations! You have successfully installed Arch Linux, but it's not quite usable yet. While it is possible to use Arch in this state, it's not very useful. You can make Arch Linux work for you by following these Post Install recommendations. Be sure to log into Arch by writing root for the login and whatever password you set before. These are reccomendations. You are free to do whatever you want. If you want an even more comprehensive set of instructions, see the decently written Arch Wiki. Arch Wiki Post Install Recommendations

2.1. Starting the Network.

It's not really possible to proceed making this system yours without starting the network. Ethernet should work without the below steps.

  1. Enable the NetworkManager service using systemctl enable NetworkManager. This command will make NetworkManager start when the system is restarted.
  2. Start NetworkManager using systemctl start NetworkManager. If you don't enable it in the previous step, then NetworkManager won't start on startup and will need to be started every time with this command.
  3. Now that NetworkManager is running, run nmcli device wifi connect <network name> password <password>, replacing <network name> and <password> the name of the network (the SSID) and with the password respectively. You can list networks using the command nmcli device wifi list.

2.2. Making a user

It is unsafe to use the root user (similar to a Superuser in Windows) for everyday tasks. More over, you can't use remote access if the only user is the root user.

  1. Run this command to make a new user. useradd -m -G wheel -s username.
  2. Be sure to replace username with a name of your choice. It can be anything, but I recommend that you give it no capital letters, no numbers, and no special characters. The argument -m creates a folder under /home/username where all your files will be stored. -G adds your user to the group wheel which will be needed for using sudo.
  3. Using nano, edit /etc/sudoers. Using the keybind CTRL+W, find the line %wheel ALL. Uncomment that line by getting rid of the trailing space and the #. Make sure the line you're editing has this written above it. ## Uncomment to allow members of group wheel to execute any command
  4. do passwd username replacing username with your username and set a new password. Make it different from the Root password for security!
  5. log out using logout then log in using your username and password.

2.3. Setting up a Desktop Environment

There are several choices when it comes to getting a desktop environment. Most of them come paired with a Desktop Manager, which you can think of as a lock screen, except in Linux, their purpose is more than to just protect your laptop from access. In Linux, as the name suggests, a Desktop Manager is used to manager and switch between several Desktop Environments (DE's for short. DM is short for Desktop Manager in this context). Without a Desktop Manager, it's not that easy to start a Desktop Environment. Lets first get a Desktop Environment.
Here, you have several options.

Easy - GNOME (gnome)

Gnome is a very simple desktop Environment and super modern too. It's not very customizable, but it is plenty if it's your first Desktop Environment. Install it using sudo pacman -S gnome. GNOME automatically installs GDM, which is a Desktop Manager. It works out of the box.

  • DM -> gdm
  • DE -> gnome

Easy - Budgie (budgie)

Budgie is a DE based on GNOME, but offers more customization over GNOME. They have similar underlying frameworks and work great when paired with GNOME apps. Install it using sudo pacman -S budgie. Some extra tools can be installed which make the Budgie experience better. Install papirus-icon-theme materia-gtk-theme arc-gtk-theme and budgie-extras along with budgie by just adding those names one by one, separated by a space to the above command. Budgie can be customized using the in-built app called Budgie Desktop Settings. It also comes with a side panel called Raven, which can be thought of as a notification shade for desktops. It does not come with a DM, but I suggest installing GDM by running sudo pacman -S gdm. It is my personal suggestion that you give Budgie a try, as I have been using it for a while and I love it.

  • DM -> Any, but GDM is reccomended (gdm)
  • DE -> budgie

Medium - KDE (plasma)

KDE, also known as KDE Plasma can offer a Windows 10 like experience. Don't let it deceive you! It's feature rich and super customizable. If you want a feature rich experience, KDE Plasma is the way to go. Compared to the previous two, KDE Plasma is leagues ahead in terms of customization. Note : Not recommended for low end devices due to high power usage. It comes with it's own DM called SDDM. Install it using sudo pacman -S plasma. An additional package called plasma-applications includes various applications that will come in handy. Install it using sudo pacman -S plasma-applications

  • DM -> sddm
  • DE -> plasma

Hard - Hyprland (hyprland)

Hyprland is the DE you go for if you want to try Ricing. I would not suggest it for anyone's intro into Linux. It's super fun to use, and uses a tiling manager, and it allows you to customize it till the ends of the earth. If you want to try an easier version of Hyprland, check out end-4's Dots Hyprland. It mostly works out of the box, and if you want to dabble your hands in Ricing, this is the place you start. It does not come with a DM, so feel free to use whatever you wish. GDM is my suggested one. Install it using sudo pacman -S hyprland

  • DM -> Any but I recommend gdm
  • DE -> hyprland

Don't forget to install a terminal!

Don't forget to install a terminal, or else your system will be nearly unusable.

  • If you installed GNOME or Budgie, go with Gnome Terminal. Install it using sudo pacman -S gnome-terminal
  • If you installed KDE Plasma, go with Konsole. Install it using sudo pacman -S konsole
  • If you installed Hyprland, go with Kitty. Install it using sudo pacman -S kitty

2.4. Starting your DM.

The easiest way to start a DE is by calling it from a DM.

  • If you installed GDM, run sudo systemctl enable gdm
  • If you installed SDDM (along with KDE Plasma), run sudo systemctl enable sddm
  • If you installed any other DM, then run sudo systemctl enable <dm name> replacing <dm name> with the shorthand name of the DM you installed.
    Run sudo systemctl start <dm name> with the name of the DM you installed.

2.5. User Directories

This is optional, but recommended for a well functioning system.
In your terminal, run ls ~. ~ is a shortcut signifying your user's main directory (Located under /home/<username>)
If your output looks a bit like this
`Documents Games Pictures Templates Desktop Downloads Music Public Videos

Then skip this step. If it does not, follow the instructions.

  1. Run sudo pacman -S xdg-user-dirs
  2. After it's done, run xdg-user-dirs-update
  3. Run ls ~ and compare the output. You should have a nice set of directories now!

2.6. Firewall

While it is not needed, it's a good to have. A firewall greatly increases your internet security.

  1. Install ufw and gufw using sudo pacman -S gufw ufw
  2. Once it's done, you should have an app called Firewall Configuration. Open it, and give it your password if it asks for it.
  3. Click the toggle "Status" and turn it on.
  4. Set Incoming to "Reject" and Outgoing to "Allow". This is enough for most users.
  5. You can set specific rules in the Rules tab, in case some app is behaving weirdly with the firewall. (Eg. P2P connections not working, not able to torrent, not able to scan for networks, not able to scan for other devices etc.)
  6. The Report tab shows all connections that are being made, the protocol and the port from which it's connecting from. If it's highlighted in red, that means it's being blocked by the firewall.
  7. The Log tab shows all modifications to the firewall profile.

2.7. Install YAY.

YAY (Yet Another Yoghurt) is an alternate package manager, similar to pacman which allows you to install packages from pacman and the AUR, which is a repository of user made packages. These are not compiled, but diverse.

  1. Create a directory called aur in your home directory using mkdir aur. This is not needed, but prevents clutter in your home directory.
  2. Run sudo pacman -S git base-devel to install some packages to compile YAY.
  3. Move to the directory aur by running cd aur.
  4. Run git clone https://aur.archlinux.org/yay.git to clone a copy of YAY.
  5. Once it's done, run cd yay
  6. Run makepkg -si to make the package and install it in one go.
  7. Install YAY.
    You can now use YAY to install any package from the AUR and compile it automatically. YAY can also replace pacman as it can pull packages from there too. In fact, YAY automates the process we just did!

2.7.1. Using YAY.

2.7.1.1. Searching for Packages
  • You can search for packages using yay <search term>. You can include multiple search terms by adding spaces such as yay foo bar
  • It will ask you for the number of the packages you want to install. You can include multiple numbers by adding spaces.
  • Do N for Clean Build unless the previous try using YAY halted unexpectedly or got corrupt.
  • Do N for Show Diffs unless you want to read the licence agreement.
  • Do N for Remove Make Dependencies unless you want to install them every time you want to compile a package (why would you want to do that?)
  • It will not ask the above questions if the package is not from the AUR, which means it'll use pacman to install them directly.
2.7.1.2. Managing Packages.
  • You can install any package directly by running yay -S <package name>, like in pacman. You can pass multiple package names by separating them with a space
  • You can remove any package by running yay -R <package name>. You can pass multiple package names by separating them with a space
  • You can remove any package along with all it's dependencies (to save space) using yay -Rs <package name>. This may not be possible for all cases due to other packages needing those dependencies.
  • You can run a full update using yay -Syu. Don't put any package names here.

2.7.2. What is YAY?

This is optional reading material.
Pacman is a very special package manager which allows front-ends to be written for it. You can even use GUI based Pacman front-ends, ditching the need for the command line! (I wouldn't suggest it.)
YAY is a frontend for Pacman that also allows you to compile and install packages from the AUR, which is a repository of user made programs and applications. It contains a lot of useful stuff, and installing from the AUR manually is a hassle. After installing YAY, using Pacman manually is mostly redundant. (It might even break YAY's functionality! ). Pacman can only install packages from the main Arch repository, which has quite a lot of packages, but the AUR has even more because it is curated by the users.

3. Optional Goodies.

These are all completely optional and based on your use case. Do not read this as a guide, rather skip to the section you need. Your system is ready to use (Really now) and feel free to ignore this whole section.

Spicing up Spotify

We'll be using a client mod called Spicetify to get the most out of Spotify. It's very powerful and supports themes.

  1. If you installed Spotify from anywhere else, remove it.
  2. Install Spotify from the AUR using yay -S spotify. It'll take some time to compile.
  3. Run this command to allow modification of the Spotify files. sudo chmod a+wr /opt/spotify; sudo chmod a+wr /opt/spotify/Apps -R This is actually two commands, seperated by the ; character, which tells them to execute one after the other.
  4. Run this command to remotely load Spicetify's installer curl -fsSL https://raw.githubusercontent.com/spicetify/spicetify-cli/master/install.sh | sh
  5. Give it "Y" when it asks whether you'd like the marketplace.
  6. It'll take some time, but when it's done, reboot Spotify.
  7. Once you're logged in, you'll see a tab called "Marketplace" in the side bar. This contains a variety of extensions and themes. Feel free to explore it!
  8. Read about Custom Apps before installing anything from the Custom Apps section. They cannot be installed from the marketplace.

Setting up an IME.

People who wish to use IMEs to type in non-English languages will need to read this guide. For this guide, it assumed that you wish to install a Japanese IME (As it's the most popular type), however other languages are compatible too. Read this guide for other languages.

IBUS for GNOME based Desktop Environments

  1. Run yay -S ibus ibus-mozc
  2. If it asks for whether you'd like Mozc or Mozc UT, select Mozc UT if you want the UT enhanced dictionary and select Mozc if you want the standard Google dictionary.
  3. Give it some time to compile. It takes a long time and a lot of power.
  4. After that's done, run ibus-setup. Go to Input Method then add both your English - English setup and another one (In the Japanese Section) called Mozc If it does not appear, restart your system
  5. Add an autostart entry (either through Budgie Desktop settings, or writing a file using the template below to ~/.config/autostart by using nano) with the command ibus-daemon -rxRd
  6. If you can't create an autostart entry, run nano ~/.config/autostart/ibus_daemon.desktop and copy paste this. (Even the [Desktop Entry] at the start!)
[Desktop Entry]
Type=Application
Name=Ibus Daemon
Description=Starts the Ibus Daemon
Exec=ibus-daemon -rxRd
  1. Modify /etc/environment using nano Use sudo
  2. Copy paste this and put it at the bottom.
GTK_IM_MODULE=ibus
QT_IM_MODULE=ibus
XMODIFIERS=@im=ibus
MOZC_IBUS_CANDIDATE_WINDOW=ibus
  1. Go to your system settings (Gnome settings or Budgie Control Center) then in the keyboard tab, add Mozc: here.
  2. Mozc should just work now! Be sure to restart your system to run the autostart entry.
  3. If you wish to modify Mozc settings, a small pop-up should appear in one corner of the screen. Alternately, in ibus-setup, under the Input Method tab, you can click preferences when Mozc is selected to see more settings.
